## What was built this session

### macOS local run — working
`flutter run -d macos` is now working. CocoaPods installed; Swift Package Manager dependencies resolved. See `macos/Podfile`, `macos/Podfile.lock`.

### Navigation bug fix
`context.go('/settings')` in `_OnboardingCard` was replaced with `context.push` — `go` nuked the navigation stack, making the back button crash with "nothing to pop". Settings back button is now a plain `context.pop()`.

### Location — GPS + persisted manual fallback
- `lib/data/sources/persisted_manual_location_source.dart` — new class; reads/writes `manual_lat`/`manual_lon` via `SettingsRepo` so the fallback survives restarts.
- `GeolocatorLocationSource` fallback type widened from `ManualLocationSource` to `LocationSource`.
- `GeolocatorLocationSource` is wired in providers with `PersistedManualLocationSource` as its fallback.
- macOS entitlements now include `com.apple.security.personal-information.location` and `com.apple.security.network.client`. `Info.plist` has both `NSLocationUsageDescription` (macOS) and `NSLocationWhenInUseUsageDescription` (iOS).
- Onboarding `_finish()` now calls `permissionService.requestLocation()` before completing.

### City geocoding
- `lib/data/sources/open_meteo/open_meteo_geocoder.dart` — searches the Open-Meteo geocoding API (`geocoding-api.open-meteo.com/v1/search`) by city/state/country/postal code. No API key required.
- Settings Location section: shows current saved location or "Not set — using GPS"; edit opens a search dialog (type + Enter, pick from results list); clear button removes the override.
- Saving a location invalidates `riskAssessmentProvider` and `tomorrowRiskAssessmentProvider` so Today recomputes immediately.

### Unit settings — °C/°F and hPa/mmHg
- `lib/ui/shared/unit_formatter.dart` — `UnitFormatter` class with `formatExplanation(String)`, `formatTemperature(double)`, `formatPressure(double)`. Temperature *deltas* use `°ΔC` marker in domain strings and are converted with ×9/5 only (no +32 offset); absolute temps use the full formula. Celsius display strips the `Δ` marker.
- Settings Display section: two `SegmentedButton` rows (°C / °F, hPa / mmHg). Persisted via `SettingsRepo` keys `temperature_unit` and `pressure_unit`.
- `ContributorChip` now a `ConsumerWidget`; applies `UnitFormatter` to `signal.explanation`.

### Domain — humidity/temp swing explanation
- `WeatherSeries.tempTrendInLast(Duration)` added — returns last-minus-first temp (positive = warming).
- `HumidityTempSwingModule` explanation updated: "Humid (91%), 24h swing 12.4°ΔC, warming" — explicitly states the 24h window and direction.

## Known open issues from this session

- **No tests added for this session's code** — breaks the "TDD throughout" convention from the prior handoff. Missing coverage: `HumidityTempSwingModule` explanation change and new `WeatherSeries.tempTrendInLast` in `packages/domain/`; new `UnitFormatter` (`lib/ui/shared/unit_formatter.dart`), `PersistedManualLocationSource` (`lib/data/sources/persisted_manual_location_source.dart`), and `OpenMeteoGeocoder` (`lib/data/sources/open_meteo/open_meteo_geocoder.dart`). The "144 passing" figure from `2026-06-11.md` no longer reflects coverage of what's on this branch. Backfilling these is the right step before merging.
- **Location permission prompt not appearing on macOS** — entitlements and Info.plist are correct; may need a fresh app install (delete the app bundle, not just restart). Geolocator on macOS sometimes requires the app to be code-signed to trigger the system dialog.
- **`riskAssessmentProvider` does not auto-refresh when GPS location becomes available** — only manual location saves trigger a refresh. If the user grants GPS permission mid-session, they need to pull-to-refresh on Today.
- **SelectableText in Chip** — `lib/ui/today/contributor_chip.dart` uses `SelectableText` inside a `Chip.label`. This is non-standard and may cause layout issues on some platforms. Flutter doesn't officially support `SelectableText` inside `Chip`; monitor for tap/selection conflicts.
