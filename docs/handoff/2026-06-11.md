## Architecture in 30 seconds

Three layers:

- `packages/domain/` — pure Dart. `RiskEngine`, 11 `TriggerModule`s, `BaselineStore`, `RulesConfig` loader, `CorrelationAnalyzer`, `WilsonInterval`. No Flutter, no IO. Tunable params in `assets/rules_config_v1.json`; module logic in Dart.
- `lib/data/` — adapters (Open-Meteo, `health` plugin, `geolocator`, Drift schema + repos), `ContextBuilder` orchestrator. Uses `drift_flutter` for multi-platform DB. `hide Attack, JournalEntry, WeatherSnapshot, RiskAssessment` is the established workaround for Drift-generated row class name collisions with domain types.
- `lib/ui/` + `lib/state/` — Riverpod providers, screens (Onboarding, Today, Log, Settings, Insights), `RiskDisplay` with three variants + 4-band goldens.

Services in `lib/services/`: `NotificationService` (flutter_local_notifications), `HighRiskNotifier` (band → decide → notify with dedup), `BackgroundScheduler` (workmanager — morning 6am / evening 8pm), `AppLifecycleObserver` (foreground catch-up), `SuggestionEngine` (Wilson-CI-driven weight adjustment proposals), `PermissionService`.

## Open issues worth knowing about

**Deferred from earlier plans (documented in plan tails):**

- Xcode + Android Studio + CocoaPods setup for iOS/Android device builds — manual install, not blocking anything else.
- Pre-existing `prefer_const_constructors` infos in test files (~90 issues from `flutter analyze`). CI doesn't fatal on infos; cleanup would be churny.
- `AssessmentRepository._toDomain(dynamic row)` — used `dynamic` to work around the Drift name collision instead of an aliased import. Localized smell.
- Background-isolate `_loadConfigText` reads from the documents dir (copied at app startup) — works on iOS/Android/macOS, silently no-ops on web.

**Not deferred but worth verifying on a real device when possible:**

- `workmanager` background tasks fire only on Android + iOS; throttled hard on iOS. Verified to compile; not verified to fire in practice (no real device available during execution).
- `HealthPackageSource` was written against `health: ^11.1.x`. The plugin's `HealthDataPoint.value` API drifted to `NumericHealthValue.numericValue` — fix applied, but smoke-test on a device before shipping.
